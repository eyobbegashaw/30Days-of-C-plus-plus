# Contributing to 30 Days of C++

We love your input! We want to make contributing to this project as easy and transparent as possible.

## How to Contribute

### Reporting Bugs
1. Use the GitHub Issues tab
2. Describe the bug in detail
3. Include code examples if possible

### Suggesting Enhancements
1. Open an issue with the enhancement suggestion
2. Explain why this enhancement would be useful

### Adding New Content
1. Fork the repository
2. Create a feature branch
3. Add your content with proper documentation
4. Submit a pull request

## Code Style Guidelines
- Use meaningful variable names
- Include comments for complex logic
- Follow C++ best practices
- Test your code before submitting

## Pull Request Process
1. Update the README.md if needed
2. Add tests for new functionality
3. Ensure all code compiles without warnings
4. Update documentation
